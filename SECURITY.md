## Security Policy

We take the security of Draft and its users seriously. This document explains how to report vulnerabilities and what you can expect from the maintainers.

### Supported Versions

Security fixes are applied to the latest code on the `main` branch. Older snapshots or forks are not guaranteed to receive security updates.

### Reporting a Vulnerability

If you believe you have found a security vulnerability in this project:

1. **Do not** create a public GitHub issue with details of the vulnerability.
2. Instead, please use one of the following options:
   - Open a private security advisory in GitHub (under **Security ▸ Advisories** for the repository), or
   - Contact the maintainers privately using the contact information provided in the repository or project profile (for example, via a listed email address).

Please include as much detail as possible:

- A description of the vulnerability and potential impact.
- Steps to reproduce the issue.
- Any relevant logs, screenshots, or proof-of-concept code.

### Response Expectations

Once your report is received, maintainers will:

1. Acknowledge receipt of your report as soon as reasonably possible.
2. Investigate and validate the issue.
3. Work on a fix and prepare a coordinated disclosure plan if appropriate.
4. Communicate progress updates until the issue is resolved.

### Responsible Disclosure

We kindly ask that you follow a responsible disclosure model:

- Allow a reasonable amount of time for a fix to be developed and deployed before publicly disclosing details.
- Avoid exploiting the vulnerability beyond what is necessary to prove its existence.

Thank you for helping keep this project and its users secure.
